## 2026-07-21 - Optimize N+1 issues in batch deletion preserving JPA Cascades
**Learning:** While Panache provides bulk delete operations (`repository.delete("id in ?1", ids)`), these operations bypass JPA lifecycle events, including `CascadeType.ALL` and `orphanRemoval = true`. In entities like `Event` where there are deep cascaded collections (e.g., allowances, tokens, reservations), bypassing lifecycle events can lead to orphaned records or foreign key constraint violations during deletion.
**Action:** To resolve N+1 query loops while preserving JPA cascades and custom authorization logic in Quarkus/Hibernate, implement a hybrid approach: perform a single `HQL` batch query with `left join fetch` to retrieve all entities and their eagerly-needed relationships into a `Map`, then iterate the input ID list to apply logic and call the iterative `repository.delete(entity)` which properly triggers cascade lifecycles.
